Octave CheatSheet v 0.1

This cheatsheet is intended to give basic forms of the main commands you might want for the Coursera Machine Learning course. I have my biases, so I include a few alternative forms of statement syntax, and reference C syntax as a presumed known thing. Them's the breaks. Many functions here have more advanced forms or can take additional parameters; there's the Octave help command for that.

Notation: names enclosed in <...> are intended to be parameters of a specific type. ^a means Control-a (written by some folks as C-a).

Octave, being a GNU production, has full readline (command history) support, which means emacs key bindings. In case you're not an emacs person, I've added a section with the most useful bindings.

I generated a pdf of this page using A3 paper settings (Chrome) and A3 Extra (Firefox). It worked for me with appropriate page breaks and being reasonably readable. Your mileage may vary.

This cheatsheet may be distributed under the GPL 2.0 or later, or the GFDL 1.3 or later, whichever works for you. There are likely to be errors, no warranty is provided, caveat emptor etc. Pull requests can go to http://github.com/apergos/octavejunk or just fork and fix up to meet your needs.

Folks with strong opinions one way or the other on Richard M. Stallman, may appreciate the following command, not listed with the rest on the cheat sheet:

fact

If you want to know what it does, you'll have to try it yourself; no spoilers here.

That is all.

Getting around

exit, quit
exit Octave
help <command>
show help for given command
pwd
show working directory from which files will be loaded
cwd <path>
change working directory to specified path
ls
display working directory contents
PS1("<stuff>")
customize the Octave prompt; see http://bit.ly/2pW4Eot (www.gnu.org docs, "Customizing the Prompt")

Variables and data

who
list all defined variables
whos
display name, type, size of all defined variables
load('<filename>.dat')
load data from specified file, into variable <filename>
csvread('<filename>.dat')
load comma-separated values data from file into variable <filename>
clear [<varname>]
unset specified variable; if no varname is supplied, unset all defined variables
save <filename> <varname>
save variable into specified file in working directory
disp(<varname>)
display value of variable to the console
disp(sprintf(<format>,<varname>))
display value of variable to the console, with specified format
global <varname>
define variable to be global (visible in any function); note that for a function to access that variable it must also contain a global declaration for it.

Basic syntax

% or #
comment (can be inline)
#{ ... }# or %{ ... }%
block comment
"stuff" or 'stuff'
string; content inside of double quotes gets backslash handling
;
optional at end of statement to suppress displaying result of statement
indentation
for human convenience only, no syntactical meaning
<varname>
displays <varname> = <current value> (unless followed by a semicolon)
printf(), scanf()
just like C
fprintf(), fscanf()
just like C. If the file handle is omitted for fprintf, it behaves just like printf.
fopen(), fclose()
just like C.

Operators

+ - * / ^
standard numerical operators
& and | or ~ ! not
standard boolean operators; '&' and 'and' are synonyms, '|' and 'or' are synonyms, '~', '!' and 'not' are synonyms.
&& ||
boolean 'short-circuit' operators that stop as soon as the truth of the condition is determined, rather than continuing to evaluate
xor()
logical exclusive or
== ~= <= >=
comparison operators
= += -= ++ --
standard assignment operators right from C; increment/decrement are both pre- and postfix

Matrix operations

[ <num1> <num2> <num3> ... ]
array (a row vector, i.e. a matrix with one row and multiple columns)
[ <num1>; <num2>; <num3>; ... ]
array (a column vector, i.e. a matrix with one column and multiple rows)
[ <num1> <num2> ...; <numa> <numb>; ... ]
matrix with as many rows and columns as the elements specified
+ - *
matrix addition, subtraction, multiplication
inv(<matrix>)
inversion of the matrix (will produce a warning and a result full of 'Inf' if the matrix is degenerate)
pinv(<matrix>)
pseudo-inversion of matrix, see http://bit.ly/2qI7Vox (Wikipedia, Moore-Penrose pseudoinverse)
<matrix>'
transpose of vector or matrix

Matrix functions

length(), size()
number of columns of a vector or matrix; rows and columns of a matrix
rand()
pseudorandom number between 0 and 1 exclusive
eye(<num>)
identity matrix of <num> rows, cols
zeros(<rows>,<cols>)
zero-filled matrix of specified size
ones(<rows>,<cols>)
matrix of specified size, filled with ones
rand(<rows>,<cols>)
matrix of specified size, filled with pseudorandom numbers between 0 and 1
magic(<number>)
'magic square' of given size (rows, columns and diagonals must have the same sum), useful for generating test data

Indexing, ranges

<matrix>(<num1>,<num2>)
selects the element from the num1 row and the num2 column
<matrix>(<num>,:)
selects row <num>
<matrix>(:,<num>)
selects column <num>
<matrix>([<num1>,<num2>,...],:)
selects the specified rows
<matrix>(:,[<num1>,<num2>,...])
selects the specified columns
<matrix>(:)
convert matrix into a column vector by concatenating each column from the matrix in order
[<num1>,<num2>,...; <numa>, <numb>,...; ...]
matrix with row one = <num1>, <num2> etc, row two = <numa>, <numb>, etc, and so on
<matrix>(:,<number>) = [...]
replace the specified column in the matrix with the specified array

Control statements

for <condition>, ... end
your standard for loop
while <condition>, ... end
your standard while loop
do ... until <condition>
executes the command block until the specified condition is met
if <condition>, ... elseif <condition>, ... else ... end
standard if statement; note that elseif is one word.
break
break from the innermost do, while, for loop.
continue
skip to the end of the innermost do, while, for loop and continue on.

Command history

^p, ^n
display previous/next command
^f, ^b
move forwards or backwards one character on command line
^a, ^e
move to the beginning (^a) or the end (^e) of the command line text
^r, ^s
search backwards (^r) or forwards (^s) from a given point in your command history for the string you type next
^k, ^y
kill (delete) the rest of the line of text from cursor, or yank (insert) the last killed piece of text at the cursor
^t
The single most useful editing command ever. You typed 'hlep' instead of 'help' for the umpteenth time. Put your cursor after the 'l', do ^t. Fixed!

Standard functions

sin(), cos(), tan()
standard trig functions; for a matrix, applied to each element
log(), log10(), exp()
standard log and exponential functions; for a matrix, applied to each element
abs(), sqrt()
absolute value, square root; for a matrix, applied to each element
sum(), prod()
sum of array elements, product of array elements; for a matrix, returns array of function result as applied to each column
floor(), ceil(), round()
largest integer <= argument, smallest integer >= argument, integer closest to argument; for a matrix, applied to each element
min(), max()
minimum value of array elements, maximum value of array elements; for a matrix, returns array of min or max value from each column

Stat functions

std()
for an array, the standard deviation of the elements; see http://bit.ly/2qBFrOV (Wikipedia, Standard deviation)
mean()
for an array, the sum of the elements divided by the number of elements; for a matrix, the mean of each column, returned as an array
norm()
norm of A; for an array, square root of sum of the squares of the elements

Plotting

plot(<variable>, <function>)
produce a plot in a separate window of the variable vs the function value
hold on
keep the plot window open and use it for subsequent plots, each new plot with a different color
close
close the plot window
title(<string>)
add the given title to the top of the graph
xlabel(<string>)
put the given label below the x axis in the plot
ylabel(<string>)
put the given label next to the y axis in the plot
legend(<description>,<description>,...)
for multiple plots in the same window, display a legend that shows the given descriptions next to each graph color
figure(<number>)
open a new plot window with the given number, or, if the window already exists, make it visible/active
subplot(<num1>,<num2>,<num3>)
dividing up the plot window into a <num1> by <num2> grid for display of multiple plots in columns and/or rows, place the new plot in the <num3> position in the grid; grid positions are counted 1, 2 ... n for the first row, n+1 ... 2n for the second row, etc.
axis([<numx0> <numx1> <numy0> <numy1>])
set the lower and upper limits for the x and y axis, respectively
plot(<variable>, <function>, 'rx')
plot the points as red X's; many other styles and colors are available, see http://bit.ly/2qBHpij (www.gnu.org docs, Two Dimensional Plots)
plot(<variable>, <function>, 'MarkerSize', 10)
plot the points rather larger than normal (default size is 6)

Function definitions

function <function_name> ... end
define a function with no parameters or return value
function <retval> = <function_name> (<param1>, <param2>, ...) ... end
define a function with the given name that returns the variable retval, with the specified parameters
function [<retval1>, <retval2> ...] = <function_name> (<param1>, <param2>, ...) ... end
define a function with the given name that returns the listed return values, with the specified parameters
function <function_name> (<param1> = <defaultval>) ... end
define a function with the given name that uses the default value provided if the caller does not supply the parameter (syntax extends to all other forms of function definition)
return
explicitly return from a function; note that return values aren't 'returned' this way, they have to be assigned to the variables specified in the function definition
